## 1. 项目概述

### 1.1 项目背景

药品安全是重大的民生和公共安全问题。为响应国家药品信息化追溯体系建设要求，本项目旨在设计并实现一个覆盖药品“生产、流通、使用”全生命周期、融合人工智能技术的智能监管与追溯平台。系统包含后台管理平台与多角色移动应用（App），以实现药品来源可查、去向可追、责任可究，并提升监管效率和公众用药安全感。

### 1.2 系统目标

1. 

   **业务核心**：构建一个多角色（监管、企业、机构、公众）协同的药品全流程追溯链条。

2. 

   **智能升级**：集成AI大模型助手，提供智能问答、数据分析与报告生成能力。

3. 

   **移动延伸**：开发跨平台移动应用，满足现场检查、扫码作业、公众查询等移动化场景需求。

### 1.3 目标用户

| 用户角色           | 主要使用终端 | 核心诉求                         |
| ------------------ | ------------ | -------------------------------- |
| **系统超级管理员** | Web后台      | 系统配置、用户与权限管理         |
| **药品监管人员**   | Web后台、App | 全流程监管、现场执法、数据分析   |
| **药品生产企业**   | Web后台      | 生产信息录入、批次管理           |
| **药品经营企业**   | Web后台、App | 进销存管理、扫码作业、库存预警   |
| **医疗机构**       | Web后台、App | 药品验收、用药记录、不良反应上报 |
| **公众**           | App          | 药品真伪查询、用药安全咨询       |

### 2.1 技术架构图

采用前后端分离的微服务化架构。





复制

```
[公众/企业/监管/机构 App (Flutter)]
         |
         | (HTTPS/API调用)
         v
[ API网关 & 负载均衡 ]
         |
         | (内部通信)
         v
    [后端服务集群]
    |-- 用户权限服务
    |-- 药品追溯核心服务

    |-- 文件服务
    |-- 消息推送服务
         |

         v

    |-- MySQL (主业务数据)
    |-- Redis (缓存/会话)
```

### 2.2 后端技术栈

- 

  **核心框架**：Spring Boot 2.7.x + Spring Cloud

- 

  **安全认证**：Spring Security + JWT

- 

  **数据持久**：Spring Data JPA / MyBatis-Plus + MySQL 8.0

- 

  **缓存**：Redis

- 

  **搜索**：Elasticsearch

- 

  **AI集成**：

  - 

    **AI编排**：LangChain4j 或 Spring AI

  - 

    **大模型接口**：DeepSeek API / OpenAI API / 本地化模型 (如 Qwen)

  - 

    **向量数据库**：Milvus 或 PGVector (存储知识库)

  - 

    **Embedding模型**：BGE-M3 (中文文本向量化)

- 

  **消息队列**：RabbitMQ (用于异步任务、日志)

- 

  **监控与日志**：ELK Stack (Elasticsearch, Logstash, Kibana)

- 

  **Web管理后台**：Vue 3 + TypeScript + Element Plus + Pinia + Vite

- 

  **移动端App**：Flutter 3.x (Dart)

    **状态管理**：Riverpod

  -

## 3. 核心功能模块详述

### 3.1 后台管理平台核心模块

1. 

   **权限与用户管理**：多级角色、部门权限控制、操作日志审计。

2. 

   **药品追溯核心**：

   - 

     **生产环节**：批号管理、原辅料关联、质检录入。

   - 

     **流通环节**：采购入库、销售出库、库存批次跟踪。

   - 

     **使用环节**：医疗机构用药记录、不良反应上报。

3. 

   **监管执法**：监督检查记录、案件管理、抽检管理。

4. 

   **追溯查询**：全链条可视化查询、公众扫码查询接口。

5. 

   **统计分析**：数据看板、风险预警地图、流通报表。

### 3.2 AI大模型助手模块 (核心新增)

1. 

   **智能问答**：通过自然语言接口，回答关于政策、操作、数据的问题。

2. 

   **业务工具调用**：AI可理解指令并自动调用后台工具函数，如：

   - 

     `query_drug_trace(batchId)`：查询药品追溯链。

   - 

     `generate_circulation_report(params)`：生成统计报告。

3. 

   **知识库管理**：后台可上传法规、手册等文档，系统自动解析、向量化存储，供AI检索增强生成 (RAG)，确保回答准确性。

4. 

   **对话与记忆**：维护多轮对话上下文，提供连贯的交互体验。

### 3.3 移动端App (“药监通”) 分角色功能

- 

  **设计原则**：一个App，通过登录角色动态切换至完全不同功能界面。

- 

  **公众版**：

  - 

    **核心功能**：药品扫码真伪查询、简化追溯链查看。

  - 

    **AI集成**：24小时用药安全智能客服，可解答用法用量、禁忌等。

- 

  **企业版 (经营企业)**：

  - 

    **核心功能**：扫码完成采购入库/销售出库、实时库存查看、预警通知（近效期、低库存）。

  - 

    **AI集成**：智能合规顾问，解答GSP相关问题、操作指引。

- 

  **机构版 (医疗机构)**：

  - 

    **核心功能**：药品扫码验收、患者用药扫码记录、不良反应快速上报。

  - 

    **AI集成**：用药安全助手，扫描药品后提示禁忌；上报辅助，智能填充报告术语。

- 

  **监管版 (监管人员)**：

  - 

    **核心功能**：现场检查任务接收与执行、扫码核验追溯信息、拍照取证、违规上报、风险地图查看。

  - 

    **AI集成**：现场数据分析助手，语音询问“该企业本月入库异常”；检查报告辅助生成。

## 4. 关键API接口设计

### 4.1 通用业务API (示例)

- 

  `POST /api/auth/login`- 用户登录

- 

  `GET /api/trace/batch/{batchId}`- 追溯链查询 (详细)

- 

  `GET /api/trace/public/{code}`- 公众扫码查询 (简化)

- 

  `POST /api/warehouse/in`- 采购入库

- 

  `POST /api/usage/record`- 用药记录登记

- 

  `GET /api/stats/circulation`- 流通统计

## 6. 移动端App与后台集成要点

1. 

   **统一认证**：App与Web后台使用同一套JWT认证体系。

2. 

   **API共用**：App调用上述所有业务API及`/api/ai/chat`接口。

3. 

   **角色隔离**：后端API需根据JWT Token中的角色信息进行严格的**数据权限过滤**，确保企业用户只能访问自身数据。

4. 

   **推送通知**：通过Firebase Cloud Messaging向App推送预警、任务等消息。

5. 

   **离线能力**：关键表单（如检查记录）支持本地草稿保存，网络恢复后自动同步。

### A. 关键术语解释

- 

  **追溯码/监管码**：药品的唯一标识码，用于追踪药品流向

- 

  **生产批次**：同一生产周期内生产的药品集合

- 

  **GSP**：药品经营质量管理规范

- 

  **RAG**：检索增强生成，AI从知识库检索相关信息后再生成答案的技术
