# 药品全流程追溯监管系统 - 启动说明

本仓库包含 **后端**、**Web 前端**、**Flutter 移动端 App**，以及 Docker 基础设施（MySQL、Redis、RabbitMQ）。

---

## 一、环境要求

| 组件 | 要求 |
|------|------|
| Docker | 用于运行 MySQL、Redis、RabbitMQ |
| Java | JDK 8 |
| Maven | 3.6+ |
| Node.js | 16+（用于前端） |
| Flutter | 3.x（用于 App，可选） |

---

## 二、一键启动（推荐）

在项目根目录 `c:\dev\design` 下打开 **PowerShell**，执行：

```powershell
.\start.ps1
```

脚本会依次：

1. 使用 Docker 启动 **MySQL、Redis、RabbitMQ**
2. 在新窗口启动 **后端**（Spring Boot，端口 8080）
3. 在新窗口启动 **前端**（Vue + Vite，端口 3000）
4. 在新窗口启动 **Flutter App**（Windows 或 Chrome）

首次运行前端会自动执行 `npm install`，首次运行 Flutter 请确保已安装 Flutter SDK 并可用 `flutter run -d windows` 或 `flutter run -d chrome`。

---

## 三、手动分步启动

若需单独控制各服务，可按以下顺序操作。

### 1. 启动基础设施

```powershell
docker-compose up -d mysql redis rabbitmq
```

等待约 20–30 秒，待 MySQL 就绪后再启动后端。

### 2. 启动后端

```powershell
cd drug-traceability
mvn spring-boot:run
```

后端地址：**http://localhost:8080**

- 数据库：`localhost:3306`，库名 `drug_traceability`，用户名/密码见 `application.yml`（默认 root/root）
- Redis：`localhost:6379`
- RabbitMQ：`localhost:5672`，管理界面 `http://localhost:15672`（guest/guest）

### 3. 启动前端

```powershell
cd drug-traceability-frontend
npm install
npm run dev
```

前端地址：**http://localhost:3000**（API 通过 Vite 代理到 8080）

### 4. 启动 Flutter App（可选）

```powershell
cd drug-traceability-app
flutter pub get
flutter run -d windows
# 或: flutter run -d chrome
# 或: 连接真机/模拟器后 flutter run
```

---

## 四、仅用 Docker 跑后端（可选）

若希望后端也在容器中运行（不本地跑 Maven）：

```powershell
docker-compose up -d
```

将启动 MySQL、Redis、RabbitMQ 和 **后端**；前端和 App 仍需在本地执行上述「启动前端」「启动 Flutter App」步骤。

---

## 五、关闭与清理

- **关闭**：直接关闭各终端/窗口即可。
- **停止 Docker 容器**：
  ```powershell
  docker-compose down
  ```
- **停止并删除数据卷**（慎用）：
  ```powershell
  docker-compose down -v
  ```

## 六、常见问题

1. **后端启动报错 “Communications link failure”**  
   说明 MySQL 尚未就绪，多等一会儿再启动后端，或先执行 `docker-compose up -d mysql` 再重试。

2. **前端 404 或接口不通**  
   确认后端已成功启动且可访问：浏览器打开 http://localhost:8080 ，再检查前端是否在 3000 端口、Vite 代理是否指向 8080。

3. **Flutter 无法连接后端**  
   App 内配置的 baseUrl 为 `http://localhost:8080`。真机调试时 localhost 指向手机本机，需将此处改为电脑局域网 IP（如 `http://192.168.1.100:8080`）。

4. **AI 相关功能报错**  
   `application.yml` 中配置了 DeepSeek API Key 与 Milvus；若未配置或未启动 Milvus，AI 相关接口可能不可用，其余功能一般不受影响。
